# Лабораторная работа №2
## «Разработка веб-приложения для публикации геоданных на базе GeoServer»

### 1. Цель работы
Освоить полный цикл подготовки, публикации и визуализации геоданных с использованием PostGIS, GeoServer и веб‑клиента на базе OpenLayers. Получить навыки настройки рабочих пространств, хранилищ данных, слоёв, а также интеграции опубликованных сервисов в веб‑приложение.

---

## 2. Содержание задания
1. Развернуть инфраструктуру для публикации геоданных.
2. Загрузить пространственные данные в PostGIS.
3. Настроить рабочее пространство и хранилище данных в GeoServer.
4. Опубликовать слои.
5. Разработать веб‑страницу, отображающую опубликованный слой через протокол WMS.

Обязательные технологии при выполнении лабораторной работы:
- Docker Compose
- PostgreSQL/PostGIS
- GeoServer
- Vite
- OpenLayers

---

## 3. Порядок выполнения работы

### 3.1. Развертывание через Docker Compose
1. Создайте файл docker-compose.yml:
```
services:
  postgis:
    image: postgis/postgis:16-3.4
    container_name: postgis
    environment:
      POSTGRES_DB: gis
      POSTGRES_USER: gisuser
      POSTGRES_PASSWORD: gispass
    volumes:
      - ./init-db:/docker-entrypoint-initdb.d
    ports:
      - "5432:5432"

  geoserver:
    image: kartoza/geoserver:2.24.2
    container_name: geoserver
    environment:
      GEOSERVER_ADMIN_USER: admin
      GEOSERVER_ADMIN_PASSWORD: geoserver
    volumes:
      - ./geoserver_data:/opt/geoserver/data_dir
    ports:
      - "8080:8080"
    depends_on:
      - postgis
```
2. Запустите контейнеры через
```
docker compose up -d
```

или с использованием среды разработки

3. Проверка доступности
Подключитесь к базе данных с использованием DBeaver или другого инструмента администрирования баз данных.
Учетные данные для подключения задаются в docker-compose: gisuser/gispass.

Проверьте доступность базы данных `gis`.

Откройте в браузере страницу: http://localhost:8080/geoserver

Авторизуйтесь с использованием учетных данных: admin/geoserver

---

### 3.2. Загрузка данных в PostGIS
1. Импорт
На данном этапе необходимо загрузить данные, полученные при выполнении первой лабораторной работы, в PostgreSQL/PostGIS.

Для импорта сырых данных через консольную команду GDAL `ogr2ogr` выполните:

`ogr2ogr -f "PostgreSQL" PG:"host=localhost user=gisuser dbname=gis password=gispass" input.geojson -nln tmp_layer -nlt PROMOTE_TO_MULTI -lco GEOMETRY_NAME=geom`

где:

- input.geojson - имя входного файла
- tmp_layer - наименование временной таблицы базы данных для загрузки данных

2. Преобразование данных с использованием SQL
Целью данного этапа является разделение входного набора данных на отдельные слои.
Необходимо создать в базе данных отдельные таблицы для базовых объектов:
- здания (полигональные объекты)
- дороги (линейные объекты)
- POI (точечные объекты)

Объекты должны быть загружены в отдельные таблицы базы данных через SQL-запросы.
К примеру, для зданий запрос для создания таблицы может выглядеть следующим образом:
```sql
select *
into buildings
from tmp_layer
where building is not null
```

3. Создание ключей и индексов
Создайте для добавленных таблиц Primary Key на основе одного из полей.

Добавьте для всех созданных таблиц пространственные индексы. Для этого используйте запросы следующего вида:
```sql
create index idx_buildings_geom
on buildings
using gist (geom);
```

4. Подготовьте SQL-скрипты для сидирования базы данных.
Создайте в папке init-db скрипты для создания структуры базы данных с нуля.
Для этого создайте файл с расширением sql, который будет содержать:
- DDL команды для создания структуры базы данных
- Результаты выгрузки данных в виде INSERT-команд

### 3.3. Настройка GeoServer

1. Создание рабочего пространства

Перейдите на страницу http://localhost:8080/geoserver

Авторизуйтесь и выберите в разделе "Данные" пункт "Рабочие области" (Workspaces).

Рабочая область определяет условное пространство имен для источников данных и слоев. Нажмите "Добавить новую рабочую область".

Задайте следующие поля:

Name: gis

Namespace URI: https://ssau.ru/gis

Сохраните изменения.

2. Создание хранилища данных
Создайте подключение в PostgreSQL/PostGIS.
Для этого выберите в меню: Хранилища -> Добавить новое хранилище -> PostGIS (Stores → Add new Store → PostGIS)

Укажите параметры подключения:

host: postgis

database: gis

schema: public

Также укажите поля: Наименование источника данных, user, passwd.

Сохраните изменения.

3. Публикация слоев

Опубликуйте слои для всех созданных в базе данных слоев.

Для этого выберите в меню: Слои -> Добавить новый слой.

Укажите хранилище данных и таблицу.

В разделе охваты нажмите на "Вычислить по данным". В результате должны быть проставлены размеры ограничивающих прямоугольников.

Сохраните изменения.

### 3.4. Разработка клиента для визуализации слоев

1. Создание приложения с использованием Vite

Перейдите в папку репозитория и создайте папку client. Выполните команду

```
npm create vite@latest
```

Укажите название проекта и фреймворк на ваш выбор.

Удалите лишние файлы из проекта.

Оставьте в разметке тэги `main` и один вложенный `div`

2. Добавьте OpenLayers

Выполните команду:

```
npm install ol
```

Подключите карту для отображения в вашем проекте. Для этого можно использовать следующий js-код:

```JavaScript
import 'ol/ol.css';
import Map from 'ol/Map';
import View from 'ol/View';
import TileLayer from 'ol/layer/Tile';
import OSM from 'ol/source/OSM';

const map = new Map({
  target: 'map',
  layers: [
    new TileLayer({
      source: new OSM()
    })
  ],
  view: new View({
    center: [0, 0],
    zoom: 2
  })
});
```

В данном случае карта будет встроена в `div` с идентификатором `map`.

Запустите приложение с использованием

```
npm run dev
```

Проверьте отображение карты.

3. Добавьте слои из GeoServer

Добавьте слои на карту с использованием кода следующего вида:

```JavaScript
import ImageLayer from 'ol/layer/Image';
import ImageWMS from 'ol/source/ImageWMS';

const wmsLayer = new ImageLayer({
  source: new ImageWMS({
    url: 'http://localhost:8080/geoserver/gis/wms',
    params: {
      LAYERS: 'gis:buildings',
      TILED: true
    },
    ratio: 1,
    serverType: 'geoserver'
  })
});

map.addLayer(wmsLayer);
```

Проверьте отображение слоев на карте
